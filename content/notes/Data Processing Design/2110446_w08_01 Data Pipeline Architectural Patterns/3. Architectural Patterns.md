# Big Data Analytics Architecture

ภาพรวมคือ เราส่ง data จาก input (sources) ไป output (data access) เพื่อนำไปใช้ประโยชน์ต่อไป แบ่งเป็น 5 ส่วน
1. Data Sources: แหล่งที่ได้ข้อมูลมา อาจจะเป็น logfile, sensors, RDBMS, etc.
2. Data Collection: การทำ Ingestion มากองไว้ที่ถังรวม (Data lake)
	1. Stage: การเปลี่ยนสภาพของ data และนำไปเก็บในถังเก็บ
		- **Data Ingestion:** Collecting data from various sources. 
		- **Data Transformation:** Cleaning, formatting, and converting data into a usable format. 
		- **Data Loading:** Storing the processed data in a data warehouse or data lake.
	2. Channel: เส้นทางการส่งของ
		- **Queues:** Data is placed in a queue and processed when the next stage is ready. 
		- **Streams:** Data flows continuously from one stage to the next. 
		- **Memory:** Data is passed directly from one stage to the next in memory.
3. Data Processing: คำนวณของรายวัน สัปดาห์ เดือน ตามที่ต้องการ
4. Result Storage: ถังที่เก็บ data พร้อมใช้ในการ analysis, presentation
5. Data Access: การนำ data จาก Result Storage ไปใช้ เช่น reports, service, etc.

# Lambda Architecture

Idea: data บางส่วนต้องถูก process ก่อนจะนำไปเก็บ หรือต้องประมวลผล real-time

Components:
1. Batch Layer: ใช้จัดเก็บข้อมูลและ/หรือประมวลผลแบบ batch เหมาะกับการคำนวณของจำนวนมากหรือคำนวณย้อนหลัง 
2. Speed Layer: data บางส่วนจะถูก duplicate มาใช้ในนี้เพื่อนำไปใช้ทันที (Stream Process)
3. Serving Layer: ส่วนที่สามารถดูข้อมูลที่ process แล้ว (ทั้ง Batch View, Real-time View)

Pros:
1. รองรับทั้ง batch, real-time processing
2. ทนต่อความผิดพร่อง สามารถคำนวณซ้ำได้บน batch process
3. ได้รับข้อมูล real-time ผ่าน Speed Layer

Cons:
1. ซับซ้อนเพราะต้องเขียนโค้ดทั้ง batch, real-time processing
2. ต้องเช็ค consistency จากข้อมูล batch กับ speed layer เวลารวมเข้าด้วยกัน
3. มี operation overhead ที่ DevOps ต้องมาจัดการ (setup Hadoop, Kafka และ Spark streaming)

# Kappa Architecture

Idea: Batch Layer เนี่ยมันก็ทำบน Real-time Layer ไปเลย ให้ทุกอย่างทำบน tech stack เดียว

Components:
1. Real-time Layer: ประมวลผลทุกอย่างบน Stream Process แบบ real-time
2. Serving Layer: ส่วนที่สามารถดูข้อมูลที่ process แล้ว

Concepts:
1. Real-time data คือทุกอย่าง batch เป็น subset ของ real-time
2. Raw data ต้องไม่เปลี่ยนแปลง
3. raw data stream อันเดียวสามารถมองได้หลายมุม (multiple views)
4. ใช้ data processing framework อันเดียวทำทั้ง batch, real-time
5. Reprocessing ตอนไหนก็ได้ เพราะ raw data ไม่เปลี่ยน

Pros:
6. Simple เข้าใจง่าย
7. Unified Architecture โค้ด process อยู่ที่เดียว
8. Latency ต่ำ เพราะ real-time
